SensIDL Outlook


The following graphic gives an outlook regarding the general usage and application process.
  1. Toolchain
    SensIDL toolchain is mainly comprised of the underlying metamodel (description language), the textual editor allowing user-friendly usage and the specific code generators.
  2. Stakeholders
    SensIDL addresses two directions of sensor-based network systems development. Both the sensor and receiver developers are able to use the SensIDL toolchain to profit from it. As the sensor developer uses the dedicated editor to describe and document sensor-specific data structures and later generates the requested executable code, that runs on actual devices; the receiver developer can resort to already formaly documented sensor descriptions and by using necessary generators generate the necessary executable code.
  3. Physical devices
    There is no direct limit to the physical devices, that can run the executable code, that is generated, as long as they are able to understand the defined data structures and speak the same language. A sensor developer can define a complex data structure for his C-based sensor device, and lets the corresponding code generator generate the necessary code, e.g. "structs". The receiver developer can use the same sensor data description to generate e.g. Java code for the Android application, that uses the information gathered from the sensor.
